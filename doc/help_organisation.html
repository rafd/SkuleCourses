Organisation of the Skule Courses Code Base

The following are the root directory structure of the code base. If it is not familiar to you, refer to the section on PHP and Symfony.

config

  • databases.yml

    The database settings are in this file. You need to change the dbname, username and password to your local settings, or to the staging/production server settings.

  • skuleGlobal.yml

    This file stores all the global settings to be used in the apps. Currently, the file is split into three sections:

    1. mail_notification: in the production environment of the courses app, a mail notification is sent every time a PHP error occurs. In both the development and production environment, a mail notification is sent every time a comment or exam has been submitted. Change sender/receiver parameters to suit the circumstances.
    2. global: Currently not in use by any app.
    3. applet_uploader: Sets the authentication for the Java Applet based multifile uploader used in adminexam by the courses app.

lib

  • model

    This directory stores the Propel database models. The om and map directories are automatically generated by Propel and therefore should not be modified. All other files under this directory are inherited from the base classes and can be customized to suit the development needs.

    Refer to the section on Skule Courses Database Layer if you are unfamiliar with the database organization.

  • SwiftMailer

    This is a third party package used for sending mails through SMTP. SwiftMailer supports ttl and ssl.

    See here for a sample snippet on how to use SwiftMailer to send an email.

  • functions.class.php

    This class only contains static methods. These methods are all used to facilitate the development of Skule Courses globally, serving as helper functions.

  • skuleGlobalConfigHandler.php

    Defines how skuleGlobal.yml should be interpreted.
